# TikTok 反爬解决方案

## 问题描述
运行一段时间后出现：
- 详情页有效数据从 30/30 变成 0/30
- 所有请求都返回空页面
- 浏览器被 TikTok 检测为自动化工具

## 根本原因
TikTok 有非常严格的反爬机制，能够检测到:
1. 连续批量的自动化请求
2. Selenium 浏览器的自动化特征
3. 过于规律的访问间隔

## 修复方案（已修改代码）

### 1. 降低批次大小
- 原: `DETAIL_BATCH = 30` → 现在: `DETAIL_BATCH = 5`
- 每次只处理5个广告，减少批量访问被检测的风险

### 2. 增加随机延迟
- 每个详情页抓取前随机延迟 2-8 秒
- 每处理5个广告后额外休息5-10秒
- 模拟真人操作的不规律性

### 3. 改用防检测浏览器
- 优先使用 `undetected-chromedriver`（需单独安装）
- 该工具专门用于绕过浏览器检测
- 若未安装则回退到普通 selenium

## 额外优化建议

### 1. 安装 undetected-chromedriver（推荐）
```bash
pip install undetected-chromedriver
```

### 2. 更换 IP 地址
- 重启路由器使用新的 IP
- 使用代理服务器（更高级）
- 避免固定 IP 被拉黑

### 3. 等待一段时间再抓取
- TikTok 的临时封禁通常 1-2 小时后会自动解除
- 建议暂停抓取一段时间后再重试

### 4. 降低整体抓取速度
```python
# 可以继续增加延迟时间
DETAIL_WAIT = 10
```

## 长期防爬策略

### 1. 分布式抓取
- 使用多台机器同时抓取
- 每台机器使用不同 IP

### 2. 使用浏览器指纹插件
- 每次请求使用不同的浏览器指纹
- 模拟真实用户的设备差异

### 3. 使用 API 替代
- 查找 TikTok 的 API 接口
- 通过 API 直接获取数据（速度更快，更稳定）

### 4. 行为模拟
- 增加随机鼠标移动
- 模拟用户的页面滚动
- 随机点击页面其他元素

## 效果预期
修改后应该能够:
- 避免被 TikTok 检测为自动化工具
- 保持较高的详情页抓取成功率
- 持续稳定地获取数据

如果还是不行，建议联系技术人员进行更高级的反爬处理。
